`Engine`（`src/engine/conversations.ts`）是 pico 跳动的心脏。它坐落在
[web 传输层](carto:web) 与 `@oh-my-pi/pi-coding-agent` SDK 之间，
持有活跃的 `conversationId → session` 映射，并把原始的 SDK 事件转化为
一个紧凑的、供订阅者渲染的流。浏览器看到的一切——流式回复、一个快照、
一个标题的出现——都是 `Engine` 的决定。

它依靠其子模块完成实际工作：[会话](carto:sessions)
负责供给代理会话，[注册表](carto:registry) 持久化元数据，
[消息与标题](carto:messages) 归一化并命名会话。

## 心智模型

**惰性、可订阅、自我回收。** 在有人对某个会话发起 prompt 或订阅它之前，
它没有活跃的会话；Engine 会按需打开一个，把它的 SDK 事件流桥接一次，
将事件扇出给订阅者，并在它闲置且无人观看足够久之后再次关闭它。

### 让它可测试的接缝

`Engine<S extends SessionLike>` 是泛型的，且只依赖抽象，而非
具体的 `Sessions` 类：

- `SessionsPort<S>` —— `get`/`open`/`close`/`isPending`（`src/engine/conversations.ts:114`）。
- `EngineDeps<S>` —— `{ db, sessions, autoTitle, onTitleSettled? }`
  （`src/engine/conversations.ts:134-139`）注入会话池、LLM
  自动标题函数，以及分支重命名钩子。

这就是为什么测试可以用假的（fake）会话来驱动 Engine。

### TurnEvent —— 扇出的词汇表

订阅者会收到三种事件之一（`src/engine/conversations.ts:123-132`）：

- `snapshot` —— 完整的窗口化消息列表 + 上下文用量 + `hasMore`。
- `delta` —— 流式传输过程中的一条进行中的消息更新。
- `title` —— 一个新的临时标题或最终标题。

`subscribe(id, cwd, mode, listener)` 会在两种**订阅模式**之一下
注册一个 listener（`src/engine/conversations.ts:121`）：

- `live` —— 随着回复流式传输，接收每一个 `delta`。
- `settled` —— 仅当回合**不**处于流式传输状态时接收快照，即一个
  合并后的最终视图。

`dispatch` 强制执行这一点：一个 `message_update` 变成只发给
`live` 订阅者的 `delta`；任何其他事件都会构建一个快照，始终发给 `live`，
但仅在非流式传输时才发给 `settled`（`src/engine/conversations.ts:383-404`）。

## 回合生命周期

`prompt(id, cwd, text)`（`src/engine/conversations.ts:246-299`）是一个回合的
主干：

1. `ensureOpen` → `get` 会话，在 `withContext({ conversationId,
   workspaceId })` 内部运行。
2. **在回合之前播种一个临时标题**（`:266`），让侧边栏
   即时更新。
3. 递增 `activeTurns`（`:267`），运行 SDK 回合。
4. 成功时调用 `emitSettled`（`:277`）；如果这是第一个回合，
   `autoTitleFromReply` 通过 LLM 升级标题（`:279`）。
5. 在 `finally` 中递减 `activeTurns` 并 `touch` 活动时钟（`:288-294`）。

`abort()` 转发给 `session.abort`。`record()` 注入一条非回合的自定义
消息（例如一条斜杠命令的结果）并重新广播一个快照。

### 标题

`seedProvisionalTitle` 仅在存储的标题仍为 `null` 时触发——它
计算一个即时的首行标题，并广播一个 `title` 事件。
`autoTitleFromReply`（`:426`）随后拉取助手回复文本，构建一个
标题上下文，调用注入的 `autoTitle`，写入最终标题，同步
SDK 会话名称，并触发 `onTitleSettled` 钩子（它会重命名 git
分支——参见 [工作树](carto:worktrees)）。

## 空闲回收

会话代价高昂，所以 Engine 会回收它们。`start(SWEEP_MS)` 装配一个
unref 的定时器（`src/engine/conversations.ts:449`）；`sweep` 释放每一个
**可回收（reapable）**会话。当一个会话**未被观看，且不繁忙，
且空闲超过 `IDLE_MS`** 时它可回收（`:498`）：

- `IDLE_MS = 10 分钟`，`SWEEP_MS = 1 分钟`（`:146-147`）。
- **被观看（viewed）** = 有 ≥1 个订阅者（`:505`）。
- **繁忙（busy）** = `activeTurns > 0` 或有待处理的 open 或正在流式传输，
  或有异步唤醒工作——正在运行的作业或排队的投递（`:509-517`）。

`release` 取消订阅 SDK 桥接，通过 port 关闭会话，并
清理映射（`:467`）。`releaseIfIdle` 是当观看者
离开时使用的手动守卫（参见 [WebHub](carto:web-hub)）。

## 上下文用量

`computeContextUsage(session)`（`src/engine/conversations.ts:90-108`）读取
会话的 token 计量，并生成一个 `ContextUsageInfo`：总 token 数、
上下文窗口、一个百分比、一个分类明细（系统提示、工具、
消息……），以及成本。它随每一个 `snapshot` 一同携带，以便客户端
渲染上下文用量计量条。
