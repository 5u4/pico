title: 密钥守卫
description: 阻止密钥文件读取并从 agent I/O 中清除密钥的 agent 扩展。
---

pico 以 **无头模式且 `autoApprove: true`** 运行 agent 会话（参见
[Sessions](carto:sessions)）—— 没有人工逐个审查工具调用。这使得泄露凭据成为
真实的风险，因此本组件是 persona 关于密钥承诺背后的强制执行层。`secret-guard.ts`
是一个守卫工具边界的 agent 扩展；`redact.ts` 则是它所使用的正则清除器。

## 心智模型

**提示词请求，扩展强制执行。**[persona](carto:sessions) 告诉 agent 不要触碰密钥，
`runtime.ts` 设置 `secrets.enabled = true`，但真正的保证存在于一个提示词无法
巧言绕过的钩子中。

`secretGuard(pi)`（`src/engine/secret-guard.ts:51`）安装三个钩子：

1. **`input`** —— 在 agent 看到之前，通过 `scrub` 清除用户文本
   （`src/engine/secret-guard.ts:52`）。
2. **`tool_call`** —— **阻止**对疑似密钥路径的 `read`/`grep`，并附上解释性理由
   （`src/engine/secret-guard.ts:58-68`），因此即使被要求，agent 也无法读取私钥。
3. **`tool_result`** —— 清除返回的工具内容，仅重写发生变化的文本块
   （`scrubContent`，`src/engine/secret-guard.ts:37,70-74`）。

## 什么算作密钥路径

`isSecretPath(path)`（`src/engine/secret-guard.ts:26`）匹配：

- **基础文件名** —— `id_rsa`、`id_ed25519`、`secrets.yml`、`.npmrc`、`.pypirc`、……
- **后缀** —— `.pem`、`.key`。
- **`.env` 与 `.env.*`** —— **但排除**模板变体 `.example`、`.sample`、
  `.template`、`.dist`，它们可以安全读取。

这个模板例外很重要：`.env.example` 是文档，`.env` 是凭据，守卫必须让 agent
读取前者。

## 清除器

`scrub(input)`（`src/engine/redact.ts:20`）应用一张有序的模式表
（`src/engine/redact.ts:1`），将可识别的凭据替换为 `[REDACTED]`：

- PEM 私钥、JWT（`eyJ…`）以及 `bearer <token>` 请求头。
- GitHub PAT/token（`github_pat_`、`ghp_`/`gho_`/`ghu_`/`ghs_`/`ghr_`）。
- OpenAI / Anthropic 密钥（`sk-`、`sk-ant-`）、AWS `AKIA…`、Google `AIza…`、
  Slack `xox…`。

:::danger
`scrub` 是一张**纵深防御的网，而非保证。**它捕获已知的 token 形态；一个新颖或
经过重新格式化的密钥可能会漏过。首要防线是对密钥路径的 `tool_call` 阻止 ——
清除器只是限制那些仍然到达 agent I/O 的内容的影响半径。
:::
