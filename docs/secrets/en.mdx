title: Secret guard
description: The agent extension that blocks secret-file reads and scrubs secrets from agent I/O.
---

pico runs agent sessions **headless with `autoApprove: true`** (see
[Sessions](carto:sessions)) — no human vets each tool call. That makes leaking a
credential a real risk, so this component is the enforcement layer behind the
persona's secret promises. `secret-guard.ts` is an agent extension that guards
the tool boundary; `redact.ts` is the regex scrubber it uses.

## Mental model

**Prompt asks, extension enforces.** The [persona](carto:sessions) tells the
agent not to touch secrets, and `runtime.ts` sets `secrets.enabled = true`, but
the guarantee lives in a hook that a prompt cannot talk its way around.

`secretGuard(pi)` (`src/engine/secret-guard.ts:56`) installs three hooks:

1. **`input`** — scrubs the user's text through `scrub` before the agent sees it
   (`src/engine/secret-guard.ts:57`).
2. **`tool_call`** — **blocks** `read`/`grep` on secret-looking paths with an
   explanatory reason (`src/engine/secret-guard.ts:63-73`), so the agent cannot
   read a private key even if asked.
3. **`tool_result`** — scrubs returned tool content, rewriting only the changed
   text blocks (`scrubContent`, `src/engine/secret-guard.ts:40,75-79`).

## What counts as a secret path

`isSecretPath(path)` (`src/engine/secret-guard.ts:26`) matches:

- **Basenames** — `id_rsa`, `id_ed25519`, `secrets.yml`, `.npmrc`, `.pypirc`, …
- **Suffixes** — `.pem`, `.key`.
- **`.env` and `.env.*`** — **except** template variants `.example`, `.sample`,
  `.template`, `.dist`, which are safe to read.

That template exception matters: `.env.example` is documentation, `.env` is a
credential, and the guard must let the agent read the former.

## The scrubber

`scrub(input)` (`src/engine/redact.ts:22`) applies an ordered pattern table
(`src/engine/redact.ts:1`) that replaces recognizable credentials with
`[REDACTED]`:

- PEM private keys, JWTs (`eyJ…`), and `bearer <token>` headers.
- GitHub PATs/tokens (`github_pat_`, `ghp_`/`gho_`/`ghu_`/`ghs_`/`ghr_`).
- OpenAI / Anthropic keys (`sk-`, `sk-ant-`), AWS `AKIA…`, Google `AIza…`,
  Slack `xox…`.

:::danger
`scrub` is a **defense-in-depth net, not a guarantee.** It catches known token
shapes; a novel or reformatted secret can slip through. The primary defense is
the `tool_call` block on secret paths — the scrubber only limits the blast
radius of whatever still reaches agent I/O.
:::
