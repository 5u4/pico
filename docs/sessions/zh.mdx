本组件持有 [引擎](carto:engine) 所驱动的**活跃代理会话**。
这里存在三个关注点：一次性构建共享的 SDK 运行时
（`runtime.ts`）、安全地按会话池化会话（`sessions.ts`），以及
把 pico 的人设和用户的身份注入到每个会话的系统
提示中（`identity.ts` + `persona.md`）。

## 共享运行时

`provisionRuntime(options)`（`src/engine/runtime.ts`）在启动时构建一个 `OmpRuntime`
并把它交给每个会话。其 `build()`（`src/engine/runtime.ts:66`）：

- 初始化 `Settings` 并**强制 `secrets.enabled = true`**
  （`src/engine/runtime.ts:78`）——这是 pico 密钥隐私职责的
  运行时层面的那一半（参见 [密钥守卫](carto:secrets)）。
- 以 `online-if-uncached` 方式刷新模型注册表，**容忍失败**，
  仅发出一条警告（`src/engine/runtime.ts:82-85`）。
- 通过一条回退链解析默认模型：角色选择 `['default']` →
  第一个默认可用 → 第一个可用；仅当一个都不存在时才报错
  （`src/engine/runtime.ts:47-49`）。

整个函数返回一个 `Result`——一个无法解析出模型的运行时是
硬性启动失败（参见 [启动序列](carto:overview)）。

## 会话池

`Sessions`（`src/engine/sessions.ts`）是 Engine 所依赖的具体 `SessionsPort`。
它维护一个 `live` 映射，以 conversationId 为键保存打开的 `AgentSession`。

### 并发安全

`open()` 绝不能为同一个会话构建两个会话，也不能
泄漏一个在关闭期间打开的会话。两种机制守护这一点：

- 一个 `pending` promise 映射对同一 id 的并发 `open()` 调用去重
  （`src/engine/sessions.ts:57-62`）——第二个调用者等待第一个的 promise。
- 一个单调递增的 `generation` 计数器（`src/engine/sessions.ts:43`）在一次 open
  开始时被捕获；如果 `closeAll()` 在此期间递增了它，那么刚构建好的
  会话会被释放，并且该 open 以 `sessions closed during open` 报错
  （`src/engine/sessions.ts:70-73`）。

### 构建一个会话

`construct()`（`src/engine/sessions.ts:112`）是会话诞生之处：

- `mkdir ~/.pico/sessions/<id>`，然后**恢复最新的 `*.jsonl` 记录**
  （通过 `latestSessionFile`——过滤 `.jsonl`、排序、取最后一个，
  `src/engine/sessions.ts:145`），或创建一个全新的。
- 用共享运行时调用 `createAgentSession`，其中 `appendSystemPrompt =
  assembleAppendPrompt(loadIdentity(identityPath))`，以及 `extensions:
  [secretGuard]`。
- `autoApprove: true`，`hasUI: false` —— pico 以**无头且
  无人值守**的方式运行会话；循环中没有人来批准工具调用。

## 身份：人设 + 用户覆盖

`assembleAppendPrompt(identity)`（`src/engine/identity.ts:28-30`）构建
追加的系统提示：单独的 `persona.md`，或在用户提供了身份时
使用 `persona + "\n\n" + identity`。

- `persona.md`（`src/engine/persona.md`）在**导入时被读取一次**
  （`src/engine/identity.ts:9`）。它把 pico 定义为一个温暖、直接的个人
  助手，并陈述两条硬性密钥规则——绝不要求用户粘贴
  密钥，以及如果看到明文密钥，警告用户轮换它并
  不回显它。这些与 [密钥守卫](carto:secrets) 的强制执行相配对。
- `loadIdentity(path)`（`src/engine/identity.ts:15-26`）读取一个可选的
  `~/.pico/identity.md`；文件缺失会静默返回 `undefined`，其他
  错误则发出警告。

:::note
人设的密钥规则是一种*提示层面*的承诺。强制执行是一个
独立的、不可协商的层——这里安装的 [密钥守卫](carto:secrets) 扩展
——因为提示可以被绕过而钩子不能。
:::
