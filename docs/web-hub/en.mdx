title: WebHub
description: The central dispatcher mapping WebSocket sockets to Engine subscriptions, routing commands, and fanning events out.

`WebHub` (`src/platforms/web/adapter.ts`) is the brain of the [web
transport](carto:web). It sits between the stateless `server.ts` pipe and the
[Engine](carto:engine), translating each `ClientCommand` into Engine calls or
[registry](carto:registry) mutations and turning Engine `TurnEvent`s back into
`ServerEvent`s. The `broadcaster.ts` companion renders the workspace/conversation
sidebar tree and holds the full socket roster.

## Mental model: three registries

The hub's entire job is bookkeeping over three data structures
(`src/platforms/web/adapter.ts:51-56`):

- `viewers: Map<conversationId, Set<HubSocket>>` — who is watching what.
- `bridges: Map<conversationId, { opened, unsubscribe }>` — **one** live Engine
  subscription per watched conversation.
- `attention: Set<conversationId>` — conversations that settled while nobody was
  viewing them.

Two invariants tie them together:

1. **A socket views at most one conversation** (`ws.data.conversationId`).
2. **A bridge exists iff that conversation has ≥1 viewer** — it is ref-counted by
   the viewer-set size, and the last viewer leaving tears it down.

### Subscription plumbing

- `bridge(convId, cwd)` (`src/platforms/web/adapter.ts:399-416`) idempotently
  opens exactly one `engine.subscribe(convId, cwd, 'live', cb)` and caches it,
  rolling back on open failure.
- `attach(ws, convId)` (`:418-427`) detaches the socket first, then points it at
  the new conversation and adds it to the viewer set.
- `detach(ws)` (`:423-436`) removes the socket and, **when the viewer set
  empties, unsubscribes the bridge** and deletes both map entries — the
  ref-count teardown.
- `activate(ws, convId, cwd)` (`:444-462`) is the composite: bridge + attach +
  clear attention + send workspaces + send snapshot.

## Command dispatch

`handleCommand(ws, command)` (`src/platforms/web/adapter.ts:117-348`) is an
exhaustive switch. Highlights:

- `heartbeat` → `heartbeatAck` (`:118-121`).
- `prompt` / `abort` → require an active, existing conversation (else `error`),
  then `engine.prompt`/`engine.abort` (`:121-141`).
- `command('ping')` → runs the command, then `engine.record` with a
  `command:ping` custom message (`:143-162`).
- `select` → validates the conversation and that its workspace platform is
  `web`, then `activate` (`:188-201`).
- `create` (and the default branch) → `provisionConversation`, bridge, attach,
  broadcast, then either `engine.prompt(prompt)` or send a snapshot (`:322-352`).
- `archive` → deprovision the worktree if any, `archiveConversation`, detach all
  viewers, finalize, rebroadcast (`:284-320`).
- `updateWorkspaceCwd` → `statSync`-checks the cwd is a directory, **refuses to
  change a worktree workspace that has active worktree conversations**, and
  `sanitizeRefSegment`s the branch prefix (`:214-282`).

`PLATFORM = 'web'` (`src/platforms/web/adapter.ts:32`) gates every cross-platform
registry access, so the web hub only ever sees web workspaces.

## Event fan-out

`dispatch(convId, turnEvent)` (`src/platforms/web/adapter.ts:372-397`) maps
Engine `TurnEvent`s to viewers of that conversation:

- `title` → a workspace-tree rebroadcast (the sidebar shows the new title).
- `snapshot` → a `snapshot` `ServerEvent`.
- otherwise (`delta`) → a `stream` `ServerEvent`.

### Attention

When a conversation settles while unviewed, `engine.onSettled` fires
`onSettled` (`:68-73`), which adds the id to the `attention` set and broadcasts
it. `broadcastAttention` (`:75-81`) sends an `attention` event to **every** socket
in the broadcaster's roster; `clearAttention` removes and rebroadcasts. The
[client](carto:web-client) turns this into the `(n) pico` browser-tab badge.

## The broadcaster

`WorkspaceBroadcaster` (`src/platforms/web/broadcaster.ts`) owns the roster of
**all** connected sockets — a separate concern from the hub's per-conversation
viewer map. Its `tree()` (`src/platforms/web/broadcaster.ts:30-51`) builds the
sidebar: `listWorkspaces` mapped to `WorkspaceSummary`, resolving each
conversation's `branch` via `currentBranch(cwd)` **only** for worktree
workspaces (async, because it reads git). `broadcast(...)` builds that tree once
and emits a per-socket `workspaces` event carrying each socket's own `activeId`
and optional draft (`src/platforms/web/broadcaster.ts:71-74`).
