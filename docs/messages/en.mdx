title: Messages & titles

Two related jobs live here: turning the SDK's raw `AgentMessage[]` into the
compact `Message[]` the browser renders (`message.ts`), and naming conversations
(`title.ts`). Both feed the [Engine](carto:engine) — `message.ts` builds every
snapshot and stream frame, `title.ts` produces the provisional and auto titles.

## Message normalization

The SDK emits a flat, verbose event log; the UI wants runs. `message.ts`
bridges the gap.

### The UI message shape

`Message = { id, role: 'user' | 'assistant' | 'system', parts: MessagePart[] }`,
where a `MessagePart` is `text`, `reasoning`, or a `tool-call`
`{ toolCallId, toolName, args, result?, isError? }` (`src/engine/message.ts`).

### Coalescing into runs

`toMessages` (`src/engine/message.ts:90`) is the core transform: it **merges
consecutive assistant messages and their tool-calls into a single "run"**,
attaches each tool result to its call by `toolCallId`, maps `custom`
display-messages to the `system` role, and passes user messages through.
`collectResults` (`src/engine/message.ts:52`) builds the `toolCallId → result`
lookup it uses.

Supporting exports:

- `assistantReplyText(...)` (`src/engine/message.ts:141`) — joins assistant text
  for the auto-title input.
- `toStreamMessage(committed, stream)` (`src/engine/message.ts:161`) — builds the
  single in-progress run from the streaming tail, or `null`.

### Windowing

Snapshots are bounded, not unbounded history:

- `tailWindow(messages, limit)` (`src/engine/message.ts:180`) — the last N
  messages plus a `hasMore` flag. `SNAPSHOT_TAIL = 40` (`src/engine/message.ts:177`).
- `olderWindow(messages, beforeId, limit)` (`src/engine/message.ts:190`) — pages
  backward for the client's "load older". `OLDER_PAGE = 30` (`src/engine/message.ts:178`).

## Titles

`title.ts` produces two kinds of title on different timescales:

- `provisionalTitle(text)` (`src/engine/title.ts:41`) — **instant and
  synchronous**: the first non-empty line, truncated to `PROVISIONAL_TITLE_MAX =
  60` with an ellipsis. Used to fill the sidebar the moment a conversation
  starts.
- `autoTitle(session, text)` (`src/engine/title.ts:20`) — **async LLM upgrade**:
  calls the SDK's `generateSessionTitle`, but returns `null` (keeping the
  provisional title) when `PI_NO_TITLE` is set or the input is low-signal.
- `titleContext(prompt, reply)` (`src/engine/title.ts:11`) — formats the
  user+assistant turn into the context string `autoTitle` consumes.

The [Engine](carto:engine) injects `autoTitle` via `EngineDeps` and calls
`provisionalTitle`/`titleContext` directly; the [registry](carto:registry)'s
`setProvisionalTitle` (guarded `WHERE title IS NULL`) vs. `setConversationTitle`
split is what lets the instant title yield to the LLM upgrade without a race.
