pico is a **personal coding-agent host**: a long-lived Bun + TypeScript process
that hosts many persistent agent conversations, streams them to a browser over
WebSocket, and persists their metadata in SQLite. Each conversation drives an
`@oh-my-pi/pi-coding-agent` session — pico is the orchestration, persistence,
and transport shell around that SDK, not the agent itself.

A separate Rust daemon supervises the process for zero-downtime deploys. So
there are really two programs in this repo: the TypeScript **worker** (`src/`)
and the Rust **supervisor** (`crates/supervisor/`), joined by one readiness
handshake.

## The boot sequence

`src/index.ts` is the worker entrypoint — a top-level-`await` module that
assembles every subsystem in order, then blocks on a shutdown promise. The
order is load-bearing:

1. **Logging** — `configureLogging()` then a `boot` logger (`src/index.ts:19-20`).
2. **Config** — `loadConfig()` returns a `Result`; on error, log, dispose, exit 1
   (`src/index.ts:22-28`). See [Configuration](carto:config).
3. **Runtime** — `provisionRuntime({ cwd })` builds the shared SDK runtime once;
   error exits 1 (`src/index.ts:30-37`). See [Sessions & runtime](carto:sessions).
4. **Store** — `openDb(defaultDbPath())` opens the WAL SQLite state DB
   (`src/index.ts:39`). See [Persistence store](carto:store).
5. **Sessions** — `new Sessions(runtime, { identityPath })` (`src/index.ts:40-42`).
6. **Engine** — `new Engine({ db, sessions, autoTitle, onTitleSettled })` then
   `engine.start()` (`src/index.ts:43-65`). The `onTitleSettled` hook renames the
   conversation's git branch once a title is generated. See [Engine](carto:engine).
7. **Web** — only if `config.web.enabled`: construct a [WebHub](carto:web-hub)
   and `createServer(...)` (`src/index.ts:66-80`). See [Web transport](carto:web).
8. **Ready handshake** — if `PICO_SUPERVISOR_SOCKET` and `PICO_READY_TOKEN` are
   set, `reportReady(...)` tells the supervisor the worker is live
   (`src/index.ts:87-98`). See [Supervisor](carto:supervisor).
9. **Shutdown** — an idempotent `shutdown(signal)` stops server, engine,
   sessions, and db, then resolves an exit code; wired to SIGINT/SIGTERM
   (`src/index.ts:100-124`).

## Subsystem map

The worker splits into a domain core, a web platform, and shared infrastructure.

- **[Engine](carto:engine)** — the conversation orchestrator. Owns the live
  `conversationId → session` map, bridges SDK events into a compact `TurnEvent`
  stream for subscribers, drives the prompt/abort turn lifecycle, seeds and
  upgrades titles, and reaps idle sessions. Its sub-modules:
  - [Sessions & runtime](carto:sessions) — the agent-session pool and identity.
  - [Registry](carto:registry) — SQLite CRUD for workspaces/conversations.
  - [Worktrees](carto:worktrees) — per-conversation git worktree branches.
  - [Messages & titles](carto:messages) — SDK→UI message normalization, titles.
  - [Secret guard](carto:secrets) — blocks secret reads, scrubs secrets from I/O.
- **[Web transport](carto:web)** — the Bun WebSocket server and wire protocol.
  - [WebHub](carto:web-hub) — command dispatch and per-conversation fan-out.
  - [Web client](carto:web-client) — the React frontend and its state machine.
- **[Configuration](carto:config)** — TOML config loading with defaults.
- **[Persistence store](carto:store)** — SQLite open, migrations, schemas.
- **[Supervisor](carto:supervisor)** — the Rust deploy daemon and handshake.

## Conventions worth knowing up front

- **Errors are values.** Fallible operations return `neverthrow` `Result<T, E>`
  rather than throwing; the boot sequence branches on `.isErr()`.
- **Parse at boundaries.** Every SQLite row and every inbound WS frame is
  Zod-validated before use — DB data and browser input are never trusted raw.
- **State lives in `~/.pico/`** — config (`config.toml`), the state DB
  (`state.db`), session transcripts, logs, and worktrees all root there.
