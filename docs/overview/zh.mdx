pico 是一个**个人编码代理宿主（personal coding-agent host）**：一个长期运行的
Bun + TypeScript 进程，托管众多持久化的代理会话，通过 WebSocket 将它们流式传输到
浏览器，并把它们的元数据持久化到 SQLite。每个会话驱动一个
`@oh-my-pi/pi-coding-agent` 会话——pico 是围绕该 SDK 的编排、持久化与传输外壳，
而不是代理本身。

一个独立的 Rust 守护进程监管该进程，以实现零停机部署。因此本仓库里其实有两个程序：
TypeScript **worker**（`src/`）和 Rust **supervisor**（`crates/supervisor/`），
二者由一次就绪握手连接起来。

## 启动序列

`src/index.ts` 是 worker 的入口——一个顶层 `await` 模块，按顺序装配每个子系统，
然后阻塞在一个 shutdown promise 上。这个顺序是承重的：

1. **日志（Logging）** —— `configureLogging()`，随后是一个 `boot` logger（`src/index.ts:19-20`）。
2. **配置（Config）** —— `loadConfig()` 返回一个 `Result`；出错时记录日志、释放资源、以 1 退出
   （`src/index.ts:22-28`）。参见 [配置](carto:config)。
3. **运行时（Runtime）** —— `provisionRuntime({ cwd })` 一次性构建共享的 SDK 运行时；
   出错则以 1 退出（`src/index.ts:30-37`）。参见 [会话与运行时](carto:sessions)。
4. **存储（Store）** —— `openDb(defaultDbPath())` 打开 WAL 模式的 SQLite 状态数据库
   （`src/index.ts:39`）。参见 [持久化存储](carto:store)。
5. **会话（Sessions）** —— `new Sessions(runtime, { identityPath })`（`src/index.ts:40-42`）。
6. **引擎（Engine）** —— `new Engine({ db, sessions, autoTitle, onTitleSettled })`，随后
   `engine.start()`（`src/index.ts:43-65`）。`onTitleSettled` 钩子会在标题生成后
   重命名该会话的 git 分支。参见 [引擎](carto:engine)。
7. **Web** —— 仅当 `config.web.enabled` 时：构造一个 [WebHub](carto:web-hub)
   并调用 `createServer(...)`（`src/index.ts:66-80`）。参见 [Web 传输层](carto:web)。
8. **就绪握手（Ready handshake）** —— 若设置了 `PICO_SUPERVISOR_SOCKET` 和 `PICO_READY_TOKEN`，
   `reportReady(...)` 会告知 supervisor：worker 已上线
   （`src/index.ts:87-98`）。参见 [Supervisor](carto:supervisor)。
9. **关闭（Shutdown）** —— 一个幂等的 `shutdown(signal)` 依次停止 server、engine、
   sessions 和 db，然后解析出一个退出码；它挂接到 SIGINT/SIGTERM
   （`src/index.ts:100-124`）。

## 子系统地图

worker 划分为一个领域核心、一个 web 平台，以及共享的基础设施。

- **[引擎](carto:engine)** —— 会话编排器。持有活跃的
  `conversationId → session` 映射，把 SDK 事件桥接为一个紧凑的 `TurnEvent`
  流供订阅者使用，驱动 prompt/abort 的回合生命周期，播种并升级标题，并回收空闲会话。
  其子模块：
  - [会话与运行时](carto:sessions) —— 代理会话池与身份标识。
  - [注册表](carto:registry) —— 针对 workspaces/conversations 的 SQLite CRUD。
  - [工作树](carto:worktrees) —— 每个会话独立的 git 工作树分支。
  - [消息与标题](carto:messages) —— SDK→UI 的消息归一化与标题。
  - [密钥守卫](carto:secrets) —— 阻止密钥读取，并从 I/O 中清除密钥。
- **[Web 传输层](carto:web)** —— Bun WebSocket 服务器与线路协议。
  - [WebHub](carto:web-hub) —— 命令派发与按会话的分发扇出。
  - [Web 客户端](carto:web-client) —— React 前端及其状态机。
- **[配置](carto:config)** —— 带默认值的 TOML 配置加载。
- **[持久化存储](carto:store)** —— SQLite 的打开、迁移与 schema。
- **[Supervisor](carto:supervisor)** —— Rust 部署守护进程与握手。

## 值得提前了解的约定

- **错误即值（Errors are values）。** 可失败的操作返回 `neverthrow` 的 `Result<T, E>`
  而不是抛出异常；启动序列基于 `.isErr()` 进行分支。
- **在边界处解析（Parse at boundaries）。** 每一行 SQLite 数据、每一个入站的 WS 帧
  都在使用前经过 Zod 校验——数据库数据和浏览器输入从不被原样信任。
- **状态存放于 `~/.pico/`** —— 配置（`config.toml`）、状态数据库
  （`state.db`）、会话记录、日志和工作树全都以此为根。
