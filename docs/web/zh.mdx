title: Web 传输层
description: Bun WebSocket 服务器，以及浏览器与 worker 之间经 zod 校验的线上协议。
---

Web 传输层是浏览器与 pico 通信的方式。两个文件划定了它的边界：
`server.ts` 是轻量的 Bun WebSocket 管道（来源校验、协议升级、帧
解码），而 `protocol.ts` 是线上契约的唯一事实来源。
所有语义都向内委托给 [WebHub](carto:web-hub)；
浏览器一侧是 [web 客户端](carto:web-client)。

## 服务器：一条无状态管道

`createServer(deps)`（`src/platforms/web/server.ts:16`）构建一个 `Bun.serve`
实例，在 `/` 提供 HTML 打包产物，并处理 `/ws`。每个套接字上唯一的
状态是 `WsData = { conversationId: string | null }`，在升级时附加
（`src/platforms/web/server.ts:7`）。所有语义决策都委托
给 hub。

### 来源强制校验

`/ws` 升级强制执行**同源**（`src/platforms/web/server.ts:30-45`）：
它将解析后的 `origin` 头的主机与 `host` 头进行比较；不匹配
（或格式错误的 origin）会被拒绝，返回 `403 forbidden origin`。**缺失** origin
头则跳过检查。成功时 `srv.upgrade` 会初始化 `{ conversationId: null }`；
升级失败返回 `400`，任何非 `/ws` 路径返回 `404`。

### 帧处理

WebSocket 处理器直接委托给 hub：`open → hub.handleOpen`、
`close → hub.handleClose`，`message` 则先 JSON 解析再 `parseClientCommand`，
在调用 `hub.handleCommand` 之前**静默丢弃**格式错误或无法识别的帧（仅记录调试日志）
（`src/platforms/web/server.ts:52-74`）。丢弃坏
帧而非报错，可以防止有缺陷或有恶意的客户端把
套接字搞崩。

## 协议：两套互不相交的词汇表

`protocol.ts` 将线上通信分为**上行命令**（不受信任、经 zod 校验）和
**下行事件**（服务器编写、纯 TS 类型）。

### ClientCommand — 浏览器 → 服务器

`clientCommandSchema`（`src/platforms/web/protocol.ts:13-54`）是一个由
应用命令加斜杠 `command` schema 组成的可辨识联合。种类如下：

- 会话：`prompt{text}`、`abort`、`select{conversationId}`、
  `create{workspaceId, prompt?}`、`draft`、`archive{conversationId}`、
  `loadOlder{conversationId, beforeId}`。
- 工作区：`createWorkspace{label}`、`renameWorkspace{workspaceId, label}`、
  `updateWorkspaceCwd{workspaceId, cwd, worktree?}`。
- 存活性：`heartbeat`。
- 斜杠命令：`command{name: 'ping', text?}`（`src/platforms/web/protocol.ts:5-11`）。

`parseClientCommand(raw)`（`src/platforms/web/protocol.ts:107`）执行 `safeParse`，
失败时返回 `undefined`——正是 `server.ts` 用来丢弃垃圾的接缝。

### ServerEvent — 服务器 → 浏览器

`ServerEvent` 联合（`src/platforms/web/protocol.ts:76-105`）：

- `workspaces{items, activeId, draftWorkspaceId?}` — 侧边栏树。
- `snapshot{conversationId, messages, isStreaming, usage, hasMore}` — 完整视图。
- `older{conversationId, messages, hasMore}` — 向后的一页。
- `stream{conversationId, message, isStreaming}` — 一个流式增量。
- `error{message}`、`attention{conversationIds}`、`heartbeatAck`。

这些是受信任的 TS 类型，而非 zod schema，因为它们由服务器编写——
校验开销只花在不受信任的入站方向上。

## 端到端

一次 prompt 往返：浏览器发送 `prompt` 帧 → `server.ts` 校验并
转发 → [WebHub](carto:web-hub) `handleCommand` → [引擎](carto:engine)
`prompt` → 流式 `TurnEvent` → hub 重新序列化为 `stream`/`snapshot`
`ServerEvent` → `ws.send` → [web 客户端](carto:web-client) 将它们归约成
React 状态。
