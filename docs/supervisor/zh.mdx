supervisor（`crates/supervisor/`）是一个小型 Rust 守护进程，掌管 pico
**worker 进程的生命周期**：它生成 `bun run src/index.ts`，阻塞直到
worker 报告就绪，并在部署槽位之间热切换以实现零停机
更新与回滚。握手的 TypeScript 一侧是 `ready.ts`；
[启动流程](carto:overview)在接近末尾时调用它。

## 两个进程，一次握手

有两个程序：Rust **supervisor** 守护进程和它管理的 TypeScript
**worker**。它们通过位于 `~/.pico/supervisor/pico.sock` 的 Unix 套接字
以换行分隔的 JSON 通信。

- supervisor **一次恰好管理一个 worker**
  （`AsyncMutex<Option<WorkerProc>>`，`crates/supervisor/src/supervisor.rs:37-51`）。
- 同一个套接字既服务于操作员 CLI（`client.rs`：`deploy`、`rollback`、
  `status`、`stop`），也服务于 worker 的就绪 ping。

## 线上协议

`proto.rs` 定义了它（`crates/supervisor/src/proto.rs`）：

- `Request`（tag `cmd`）：`Deploy{path}`、`Rollback`、`Status`、`Stop`、
  `Ready{token}`（`crates/supervisor/src/proto.rs:7-14`）。
- `Response`（tag `status`）：`Ok{detail}`、`Status(StatusReport)`、`Error{message}`
  （`crates/supervisor/src/proto.rs:17-22`）。
- `ReadyAck{}` 序列化为 `{}`——正是 `ready.ts` 等待的那个 ack
  （`crates/supervisor/src/proto.rs:25`）。

## 就绪握手

这是两个进程之间的连接点。

**Worker 侧** — `reportReady(socketPath, token)`
（`src/supervisor/ready.ts:7-35`）：连接到套接字，并在连上后写入
单个帧 `{"cmd":"ready","token":...}\n`（`src/supervisor/ready.ts:26`）。当
supervisor 的 ack `data` 到达时它 resolve `ok()`，或在超时
（`READY_TIMEOUT_MS = 10s`）、套接字错误、或 ack 之前关闭时报错。

**Supervisor 侧** — 当 worker 被生成时，`spawn_and_validate`
（`crates/supervisor/src/supervisor.rs`）在 `pending_ready` 中存储一个
以新铸造 token 为键的 `oneshot` 发送端，然后以环境变量
`NODE_ENV=production`、`PICO_SUPERVISOR_SOCKET=<socket>` 和
`PICO_READY_TOKEN=<token>` 启动 `bun run src/index.ts`。随后它 `select` 于：ready oneshot 触发、
子进程过早退出、或 `health_timeout`（默认 30s）到期。`handle_conn`
把入站的 `Ready` 路由到 `signal_ready`，后者将 token 与
`pending_ready` 匹配，触发 oneshot（解除生成的阻塞），并写入 `ReadyAck`
（`crates/supervisor/src/supervisor.rs:199-208`）。一个从未按时 ack 的 worker
会被杀死，部署失败。

## 蓝绿槽位

`slots.rs` 将部署/回滚实现为 `<dir>/slots` 下的**符号链接交换**
（`crates/supervisor/src/slots.rs`）：

- `current` 与 `previous` 符号链接指向槽位目录；`current_target()`
  / `previous_target()` 读取它们（`:17-23`）。
- `promote(slot)` 把 `current → previous` 移动并将 `current` 指向新槽位
  （`:25-30`）；`swap()` 交换两者以回滚（`:32-42`）。
- `set_link` 写入一个 `.tmp` 符号链接然后重命名覆盖目标，使交换
  **原子化**（`:53-62`）。

一次部署会校验目标槽位包含 `src/index.ts`、杀死旧
worker，并 `spawn_and_validate` 新的那个；成功时 `promote`，
失败时 `recover` 回上一个槽位。始终只保留两个槽位。

## 配置与关停

`config.rs` 读取 `~/.pico/supervisor/supervisor.toml`（若不存在则全部默认）：
`socket_path`、`bun_path` 和 `health_timeout_secs`（默认 30）。`resolve_bun`
依次搜索 `$BUN_INSTALL/bin/bun`、`$HOME/.bun/bin/bun`、`PATH`，最后是裸的 `bun`
（`crates/supervisor/src/config.rs:65-82`）。

在 `SIGTERM`/`SIGINT` 时 supervisor 取消其 task tracker、清空
连接，并向 worker 发送 SIGTERM；worker 独立地捕获相同
信号并运行自己的[优雅关停](carto:overview)——这两条关停
路径是分开的，且都必需。
