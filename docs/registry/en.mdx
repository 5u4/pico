The registry (`src/engine/registry.ts`) is the **metadata source of truth**: a
set of pure functions over a `bun:sqlite` `Database` that read and write the
`workspaces` and `conversations` tables. Everything above it — the
[Engine](carto:engine), [WebHub](carto:web-hub), and [provisioning](carto:worktrees) —
goes through these functions rather than touching SQL directly.

## Mental model

**A trust boundary, not just a data-access layer.** SQLite rows are external
data, so every read is validated through `workspaceSchema.parse` /
`conversationSchema.parse` (from the [store](carto:store)) before it leaves the
registry. A malformed row throws rather than propagating a half-typed object.

The two tables form a tree: a workspace owns many conversations
(`workspaceId REFERENCES workspaces(id) ON DELETE CASCADE`). A workspace is
either a plain directory or a **worktree workspace** (has both `defaultBranch`
and `branchPrefix`) — see [Worktrees](carto:worktrees).

## Workspace operations

- `listWorkspaces(platform)`, `getWorkspace(id)` — reads.
- `createWorkspace(...)`, `getOrCreateDefaultWorkspace(...)` — the latter backs
  the web hub's "Default" workspace so a fresh install has somewhere to talk.
- `renameWorkspace(id, label)`, `updateWorkspaceCwd(id, cwd, worktree)` — mutate
  label and directory; `updateWorkspaceCwd` also flips a workspace between plain
  and worktree mode.

## Conversation operations

- `listConversations(workspaceId)` — **excludes archived**, ordered `DESC`
  (`src/engine/registry.ts:126`).
- `getConversation(id)` (`src/engine/registry.ts:148`),
  `createConversation(...)` (`src/engine/registry.ts:159`).
- `countActiveWorktreeConversations(workspaceId)` — joins to count live
  worktree conversations (`src/engine/registry.ts:216`); the hub uses this to
  refuse switching a worktree workspace's cwd while conversations are active.
- `archiveConversation(id)` — soft-delete, `WHERE archivedAt IS NULL`
  (`src/engine/registry.ts:230`).

### Provisional vs. final titles

Two title setters encode pico's title lifecycle:

- `setProvisionalTitle(id, title)` (`src/engine/registry.ts:191`) updates
  **only** `WHERE title IS NULL` and returns whether a row changed — so an
  instant provisional title never clobbers a real one.
- `setConversationTitle(id, title)` (`src/engine/registry.ts:205`) writes
  unconditionally — used by the LLM auto-title upgrade.

The `IS NULL` guard is the whole mechanism: provisional titles are best-effort
and yield to any title that already exists. See [Messages & titles](carto:messages)
for how these are produced and [Engine](carto:engine) for when they fire.
